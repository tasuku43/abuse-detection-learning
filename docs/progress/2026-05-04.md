## 学んだこと

* 本番化では model 本体より先に、label source、feature row contract、point-in-time correctness、evaluation result の保存が必要になる
* Phase 0〜7 の中心は、`feature row -> scoring_fn -> risk_score -> threshold sweep -> error analysis` の責務分離だった
* `as_of_time` が1時点だけだと、rolling window evaluation の器があっても期間差は見えない
* 複数時点の synthetic fixture があると、window ごとの metrics を観察できる
* window ごとに FP が増えると、全体 metrics だけでは見えにくい期間限定の精度劣化を見つけやすい
* random split は最初の確認には便利だが、未来データへの汎化を見るには time-based split の方が自然である
* 不正検知では攻撃パターンが時間で変わるため、過去で学習して未来で評価する形が重要になる
