# Pre-implementation Checklist

このチェックリストは、coding agent が実装前説明を省略しないためのハーネスです。

ファイル編集の前に、次の項目をユーザーに説明します。

## 必須説明

- [ ] 今回作るものを一文で説明した
- [ ] なぜそれが必要かを説明した
- [ ] このステップで何を学べるかを説明した
- [ ] 変更予定ファイルを列挙した
- [ ] 実装後に読む順番を説明した
- [ ] 動作確認方法を説明した

## 境界確認

- [ ] 実データ、顧客データ、社内データ、secret を使わないことを確認した
- [ ] scoring logic が label を見ない設計になっているか確認した
- [ ] DB、ネットワーク、本番環境に接続しないことを確認した
- [ ] 既存の未コミット変更を勝手に完成扱いしていない

## 実装後確認

- [ ] 実装したコードを読むための説明をした
- [ ] テストで何を確認しているか説明した
- [ ] 可能なら `pytest -q` を実行した
- [ ] `docs/progress/` に学びを残した
- [ ] `docs/planning/tasks.md` を更新した

## 省略禁止ルール

「次は Phase N をやります」だけでは不十分です。

最低限、次の形で説明してから実装します。

```text
今回は X を作ります。
これは Y を理解するために必要です。
変更予定は A, B, C です。
読む順番は A -> B -> C です。
確認は Z で行います。
```
