# Docs

このディレクトリは、設計、計画、作業プロセス、学習ログを分けて置きます。

## 構成

* `design/`
  * 架空の SaaS 不正検知アーキテクチャ設計書を置く
  * 実装手順や学習ロードマップは含めない

* `planning/`
  * ロードマップとタスクリストを置く
  * `docs/design/` をどう読むか、次に何を作るかを管理する

* `process/`
  * coding agent の作業手順、実装前チェックリストを置く

* `learning/`
  * Phase ごとの学び、評価フロー、総まとめ、gap analysis を置く

* `progress/`
  * 日別の作業ログを置く

## 読む順番

1. `design/`
2. `planning/roadmap.md`
3. `planning/tasks.md`
4. `learning/learning_review.md`
5. `progress/` の最新ログ
